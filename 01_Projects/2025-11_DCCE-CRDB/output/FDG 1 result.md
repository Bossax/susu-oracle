|                                              | **ความยากง่ายของภาษา**           | • คำศัพท์และคำอธิบายยากเกินไปสำหรับภาคประชาชน (ชาวบ้าน)<br><br>  <br><br>• ต้องมีความเข้าใจตรงกันทั้งผู้ให้และผู้รับข้อมูล                                                                                           |                    |
|                                              | **คุณภาพและการจัดเก็บข้อมูล**    | • ข้อมูลมีความล่าช้า ไม่ครบถ้วน และไม่ได้ตามที่ร้องขอ<br><br>  <br><br>• Format ข้อมูลที่หน่วยงานจัดเก็บ ไม่ตรงกับรูปแบบที่ต้องการใช้งาน<br><br>  <br><br>• การเชื่อมโยงข้อมูลจาก ทสจ. ไม่มี API ทำให้เชื่อมโยงลำบาก |                    |
|                                              | **กระบวนการ QA/QC**              | • การทำ QA/QC ในแต่ละกลุ่มงานไม่มี SOP (Standard Operating Procedure) กลาง<br><br>  <br><br>• ยังไม่ชัดเจนว่าใครรับผิดชอบ QA/QC ทั้งฝั่ง Input-Output                                                                |                    |
|                                              | **การแบ่งระดับข้อมูล**           | • อยากให้แบ่งข้อมูลเป็น 3 ระดับ (นโยบาย / ผู้ปฏิบัติ / ประชาชน) เพื่อการสื่อสารที่ตรงเป้าหมาย<br><br>  <br><br>• ข้อมูลสำหรับประชาชนควรแปลผลให้เข้าใจง่าย                                                            |                    |
|                                              | **การเชื่อมโยงข้อมูลต่างประเทศ** | • อยากเห็นการเชื่อมโยงข้อมูล Adaptation จากต่างประเทศ เพื่อให้เกิดไอเดียในการปรับใช้<br><br>  <br><br>• ต้องการข้อมูลกลไกการขอทุนจากต่างประเทศในแต่ละช่องทาง                                                         |                    |
| **2. การออกแบบระบบ (Platform & UX/UI)**      | **มาตรฐานการออกแบบ**             | • ควรออกแบบ Platform ตาม cycle มาตรฐานของ Adaptation (อ้างอิง GGA)<br><br>  <br><br>• อยากให้แสดงผลข้อมูลเป็น Dashboard                                                                                              | ความเห็นจาก ผอ.นิด |
|                                              | **การสื่อสารสองทาง (Two-way)**   | • ต้องการระบบ Two-way communication<br><br>  <br><br>• ควรมี FAQ และ Chatbot (อาจใช้ AI) ช่วยตอบคำถามเบื้องต้น                                                                                                       |                    |
|                                              | **การเข้าถึงและใช้งาน (UX/UI)**  | • ชาวบ้านไม่สามารถใช้งานหรือทำความเข้าใจระบบเว็บไซต์ได้<br><br>  <br><br>• ตัวระบบปัจจุบันยังไม่ User Friendly<br><br>  <br><br>• อยากเห็นหน้าตาข้อมูล (Preview) ก่อนทำการเชื่อมโยง                                  |                    |
| **3. กระบวนการทำงาน (Workflow & Comm.)**     | **ขั้นตอนการประชาสัมพันธ์ (PR)** | • กลุ่ม PR ต้องรอให้ข้อมูลลง Platform ก่อน จึงจะนำไปลง Facebook ได้<br><br>  <br><br>• สื่อที่ผลิตต้องส่งให้กลุ่มงานตรวจทานก่อนเผยแพร่                                                                               |                    |
|                                              | **ช่องทางการสื่อสาร**            | • เว็บกรมฯ: เน้นข้อมูลเชิงเทคนิค/วิชาการ/ข่าวสาร<br><br>  <br><br>• CCIC / Platform: ควรเน้นข้อมูลด้านการปรับตัว (Adaptation)<br><br>  <br><br>• การสื่อสารไปยังกลุ่มผู้ใช้ยังน้อยเกินไป                             |                    |
|                                              | **บทบาทในพื้นที่**               | • องค์กรในพื้นที่/หน่วยงานการศึกษา สามารถแปลงข้อมูลและถ่ายทอดให้ชาวบ้านเข้าใจได้ดีกว่าส่วนกลาง                                                                                                                       |                    |
